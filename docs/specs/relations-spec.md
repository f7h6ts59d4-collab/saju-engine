# 작업 명세: 합충형파해(合沖刑破害) + 공망 계산 추가

> saju-engine 레포. 원국 4기둥의 천간·지지 상호관계(합·충·형·파·해·원진)와 공망을
> 계산해 출력에 추가한다. "기둥이 살아있는지 깨졌는지"를 읽는 통변의 뼈대.
> 기존 로직 무변경, 리팩토링 구조 준수(새 모듈). 원칙은 CLAUDE.md.

## 표준: 포스텔러 만세력 호환 (명식 2개로 대조 검증)

- 명식 P: 임신·정미·경자·무인 (1992-07-23 05:20 대구, 남)
- 명식 Q: 경진·경진·계해·갑인 (2000-05-05 05:05 대구)

포스텔러 관측:
- P → 정임합(천간), 신자반합, **임무충(천간)**, 인신충, 미자해, 미자원진
- Q → 인해육합, **인진반합 2개**, **갑경충(천간) 2개**, 진진자형, 해인파, 진해원진 2개

## 검증으로 확정된 3가지 (중요)

1. **천간충이 존재한다** — 규칙: **같은 음양 + 오행 상극**. 10쌍:
   양간 갑무·갑경·병경·병임·무임 / 음간 을기·을신·정신·정계·기계.
   (P의 임무충, Q의 갑경충 2개 재현 확인.)
2. **부분형은 세지 않는다** — P에 인(시)·신(연)이 있어 인사신 삼형 중 2개가 갖춰졌으나
   포스텔러는 형을 보고하지 않음. **완전 삼형·상형·자형만 인정**, 삼형 중 2개(부분형) 제외.
3. **반합은 삼합 부분 + 방합 부분 모두 포함** — P의 신자반합은 삼합(신자진) 부분,
   Q의 인진반합은 **방합(인묘진) 부분**. 둘 다 반합으로 표기해야 관측 재현됨.

## 설계 원칙

- **겹침 허용**(신살과 동일): 같은 지지쌍이 여러 관계에 해당하면 전부 표기.
  (Q의 인해 = 육합 + 파 둘 다 보고됨. P의 미자 = 해 + 원진 둘 다.)
- **합화(合化) 판정 안 함**: "정임합이 성립한다"는 사실만 낸다. 실제로 목으로 화(化)했는지는
  월령·투출 등 학파별 조건 판단이라 엔진이 단정하지 않는다(지어내지 않기 원칙).
  합의 오행은 참고 필드로만.
- **인접/원격 표시**: 인접 = 연월·월일·일시(붙어있는 두 기둥). 원격 = 연일·연시·월시.
  명리에서 인접 작용이 강하므로 해석에 쓰인다.
- 대운·세운과 원국의 충합은 **이번 범위 아님**(다음 단계).

## 규칙표

### 천간합 (5쌍) — 참고 오행
갑기→토, 을경→금, 병신→수, 정임→목, 무계→화

### 천간충 (10쌍) — 같은 음양 + 상극
갑무, 갑경, 병경, 병임, 무임 / 을기, 을신, 정신, 정계, 기계

### 지지 육합 (6쌍) — 참고 오행
자축→토, 인해→목, 묘술→화, 진유→금, 사신→수, 오미→토

### 지지 삼합 (4국)
인오술→화, 사유축→금, 신자진→수, 해묘미→목
- 3개 모두 → **삼합**. 2개만 → **반합**(어느 2개 조합이든).

### 지지 방합 (4국)
인묘진→목(동), 사오미→화(남), 신유술→금(서), 해자축→수(북)
- 3개 모두 → **방합**. 2개만 → **반합**(방합 부분 — Q의 인진반합).

### 지지 충 (6쌍)
자오, 축미, 인신, 묘유, 진술, 사해

### 형
- **삼형**(3개 전부 있을 때만): 인사신, 축술미
- **상형**: 자묘
- **자형**: 진진, 오오, 유유, 해해 (같은 지지 2개 이상)
- ※ **부분형(삼형 중 2개)은 제외** — 검증으로 확인됨

### 파 (6쌍)
자유, 축진, 인해, 묘오, 사신, 술미

### 해 (6쌍)
자미, 축오, 인사, 묘진, 신해, 유술

### 원진 (6쌍)
자미, 축오, 인유, 묘신, 진해, 사술

### 공망 (순중공망)
일주가 속한 순(旬)의 빠진 두 지지가 공망. **연지·월지·시지**에서 판정(일지는 기준이라 제외).
- 갑자순(60갑자 0~9) → 술·해
- 갑술순(10~19) → 신·유
- 갑신순(20~29) → 오·미
- 갑오순(30~39) → 진·사
- 갑진순(40~49) → 인·묘
- 갑인순(50~59) → 자·축
> 학파 무관 표준. 스토리 의미: "채워지지 않는 결핍" — 궁성과 엮어 해석(시주 공망=자녀궁 결핍 등).

## 출력 (correctPillars에 추가)

```ts
relations: {
  type: '천간합'|'천간충'|'육합'|'삼합'|'삼합반합'|'방합'|'방합반합'
      |'충'|'삼형'|'상형'|'자형'|'파'|'해'|'원진';
  name: string;          // '정임합', '인신충', '진진자형'
  element?: string;      // 합의 참고 오행 (합화 판정 아님)
  positions: string[];   // ['연주','월주'] — 성립한 기둥들
  adjacent: boolean;     // 인접 여부(연월·월일·일시=true)
}[];
gongmang: {
  branches: string[];    // 일주 기준 공망 지지 2개, 예: ['진','사']
  positions: string[];   // 실제 공망인 기둥(연/월/시). 없으면 빈 배열
};
```
- 같은 관계가 여러 쌍에서 성립하면 각각 별도 항목(겹침 허용).
- timeUnknown이면 시주 관련 관계·공망 판정 제외.
- 히트 없으면 relations는 빈 배열.

## 검증

**명식 P (임신·정미·경자·무인)**:
- 천간합 정임합(목): [연주,월주] adjacent=true
- 천간충 임무충: [연주,시주] adjacent=false
- 삼합반합 신자(수): [연주,일주] adjacent=false
- 충 인신충: [연주,시주] adjacent=false
- 해 자미해: [월주,일주] adjacent=true
- 원진 자미원진: [월주,일주] adjacent=true
- 형·파·육합·방합: 없음 (※인신 부분형 제외 확인)
- 공망: branches ['진','사'], positions [] (히트 없음)

**명식 Q (경진·경진·계해·갑인)**:
- 육합 인해합(목): [일주,시주] adjacent=true
- 방합반합 인진(목): [연주,시주] adjacent=false / [월주,시주] adjacent=true → **2건**
- 천간충 갑경충: [연주,시주] adjacent=false / [월주,시주] adjacent=true → **2건**
- 자형 진진: [연주,월주] adjacent=true
- 파 인해파: [일주,시주] adjacent=true
- 원진 진해: [연주,일주] adjacent=false / [월주,일주] adjacent=true → **2건**
- 천간합·지지충·삼합·해: 없음
- 공망: branches ['자','축'], positions [] (히트 없음)

**명식 R (공망 검증용, 1990-02-14 06:30 서울 → 경오·무인·경술·기묘)**:
- 공망: branches ['인','묘'] (경술=갑진순), positions ['월주','시주']

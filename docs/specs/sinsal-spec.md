> 구조에 맞춰 **새 모듈(src/correction/sinsal.ts)**로 만든다. 원칙은 CLAUDE.md.

## 표준: 포스텔러 만세력 호환 (실제 명식 11개로 전수 검증)

아래 모든 규칙표는 실제 포스텔러 출력 11개 명식으로 대조·프로그램 검증됨.
검증 명식(전부 서울, 시간은 시주 경계 회피): 임신·정미·경자·무인(경,남) /
경오·무인·기유·신미(기) / 기사·임신·계축·신유(계) / 경오·신사·갑술·임신(갑) /
경오·무인·을사·갑신(을) / 무진·갑인·병오·경인(병) / 정묘·임자·정사·정미(정) /
정묘·계축·무자·병진(무) / 정묘·계축·기사·무진(기) / 기사·병자·신유·무술(신) /
무진·갑인·임자·무신(임). 성별 미지정 명식은 여성 기준으로 받음(고신/과숙 성별 규칙 확인).

## 설계 원칙: 겹침 허용

**같은 신살이 여러 번 성립하면 그대로 다 표기한다**(제거·병합 금지). 신살은 중첩될수록
기운이 강한 것으로 본다. 예: 역마살이 12신살에서 1회 + 글자그룹에서 1회 → 총 2회 표기.
각 성립을 positions(성립 기둥)로 구분한다.

## 1. 12신살 (twelve) — 혼합 기준

**월지·일지·시지는 연지(年支) 삼합 기준, 연지 자신은 일지(日支) 삼합 기준.**
> 판별 명식 2개(연·일지 삼합국이 다른 B·E)에서 연지가 일지 기준 값으로 표시됨 확인.
> 11명식 재현. 반례 발견 시 이 명세 기준으로 보정.

삼합국: 인오술(화)/사유축(금)/신자진(수)/해묘미(목).
배치: 묘지(끝 글자)의 다음 지지가 겁살, 이후 순서대로
겁살→재살→천살→지살→년살(도화)→월살→망신살→장성살→반안살→역마살→육해살→화개살.

## 2. stars (18종)

### 일간 기준 (지지에서 매칭)
| 신살 | 규칙 |
|------|------|
| 천을귀인 | 갑무경→축·미 / 을기→자·신 / 병정→해·유 / 신→인·오 / 임계→묘·사 |
| 문창귀인 | 갑→사 을→오 병→신 정→유 무→신 기→유 경→해 신→자 임→인 계→묘 |
| 문곡귀인 | (문창의 충) 갑→해 을→자 병→인 정→묘 무→인 기→묘 경→사 신→오 임→신 계→유 |
| 학당귀인 | 갑→해 을→오 병→인 정→유 무→인 기→유 경→사 신→자 임→인 계→묘 |
| 관귀학관 | 갑을→사 병정→신 무기→해 경신→인 임계→신 |
| 태극귀인 | 갑을→자·오 병정→묘·유 무기→진·술·축·미 경신→인·해 임계→사·신 |
| 정록 | 갑→인 을→묘 병→사 정→오 무→사 기→오 경→신 신→유 임→해 계→자 |
| 암록 | (정록의 육합) 자-축 인-해 묘-술 진-유 사-신 오-미 |
| 양인살 | (정록 다음 지지) 갑→묘 을→진 병→오 정→미 무→오 기→미 경→유 신→술 임→자 계→축 |
| 금여성 | (정록 두 칸 뒤) 갑→진 을→사 병→미 정→신 무→미 기→신 경→술 신→해 임→축 계→인 |
| 홍염살 | 갑→오·신 을→오·신 병→인 정→미 무→진 기→진 경→신 신→유 임→자·신 계→신 |

> 학당 임 셀은 장생 규칙(신)과 관측(인)이 달라 **관측표(인) 하드코딩**.
> 홍염 갑·을·임은 지지 2개(관측 확정). 문곡은 문창의 충 지지(자-오,축-미,인-신,묘-유,진-술,사-해).

### 글자 모양 기준
| 현침살 | 甲·辛·卯·午·未·申 — **천간·지지 모두** 판정 |

### 일주 기준 (전 기둥에서 해당 간지 탐색)
| 괴강 | 경진·경술·임진·임술·무술 (5개) |
| 백호대살 | 갑진·을미·병술·정축·무진·임술·계축 (7개) |

### 지지쌍 — 인접 기둥(연월/월일/일시)에서만 성립
| 귀문관살 | 자-유, 인-미, 묘-신, 진-해, 사-술 (5쌍. **축-오는 제외** — 판별 명식으로 확인) |

### 글자 그룹 (12신살과 겹침 허용 — 별도 표기)
| 도화살 | 자·오·묘·유 (가도화) |
| 역마살 | 인·신·사·해 |
| 화개살 | 진·술·축·미 |

> 도화·역마·화개는 12신살(twelve)에도 정식으로 들어가고, 여기서 글자그룹으로 또 잡힌다.
> 겹치면 겹친 대로 stars에 각각 표기(설계 원칙).

## 3. correctPillars 출력

```ts
sinsal: {
  twelve: { year: string; month: string; day: string; hour: string | null };
  stars: { name: string; positions: ('연주'|'월주'|'일주'|'시주')[] }[];
};
```
- timeUnknown이면 시주 판정 제외(twelve.hour = null, stars에서 시주 제외).
- 같은 name이 서로 다른 성립 근거로 여러 번 나올 수 있음(겹침 허용). positions로 구분.
- 히트 없으면 stars는 빈 배열.
